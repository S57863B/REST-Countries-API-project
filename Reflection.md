In this project I built a REST Countries application using TypeScript, HTML, and CSS. My main goal was to fetch country data from an API and display it in a clean and user friendly way. I also added features like search, filter by region, dark mode, and a detailed country page.

The beginning of the project was quite challenging. I struggled with the initial architecture, leading to several rounds of restructuring my folders and files. However, this "rough start" forced me to pause and develop a formal project plan. I began with a semantic HTML layout and basic CSS to establish the visual hierarchy. Transitioning to TypeScript, I focused on core logic: the fetch mechanism, rendering engine, and state management for themes and filters.

The most demanding part was the app.ts file and the rendering function. Ensuring that the UI updated efficiently without unnecessary re-renders required a deep dive into DOM manipulation and Type casting. Additionally, styling the application using Vanilla CSS was time-consuming but rewarding, as it sharpened my ability to handle complex layouts and dark mode transitions without the crutch of external frameworks.

Looking ahead, I have identified several areas for growth. I plan to implement a hover-feature that displays flag previews when a user focuses on neighboring countries. Furthermore, I intend to expand the filtering capabilities to include sorting by population, sub-regions, and languages. Finally, I aim to refine the UI with dynamic animations and micro-interactions to provide a more polished user experience. This project reinforced the importance of planning before coding and solidified my confidence in handling asynchronous data within a typed environment.
